## ğŸš€ Getting Started

> **Prerequisite:** You need a Google AI Studio API key with access to Gemini Flash models.

```bash
git clone https://github.com/Spoorthy1423/DevSpace_Code_Generator.git
cd DevSpace_Code_Generator/frontend
npm install
```

Create a `.env` inside `frontend`:

```bash
VITE_GEMINI_API_KEY=your_api_key_here
```

> The app currently instantiates `GoogleGenAI` directly in `Home.jsx`. Replace the placeholder with `import.meta.env.VITE_GEMINI_API_KEY` to avoid hardcoding secrets when you add the `.env`.

Start the dev server:

```bash
npm run dev
```

Open the provided localhost URL. Vite will choose a free port automatically and prints it in the terminal (e.g. `http://localhost:5173` or `5175`).

Build for production:

```bash
npm run build
npm run preview   # optional local preview
```

## ğŸ§  Key Implementation Notes

- **Error Handling:**  
  - 429 (rate limit) responses trigger a delayed automatic retry.  
  - 503 (model overloaded) surfaces a toast; manual retry is currently required.  
  - Browser extension warnings (e.g. â€œlistener indicated an asynchronous responseâ€¦â€) stem from injected scripts and can be ignored or tested in incognito mode.

- **Preview Handling:**  
  The inline iframe is sandboxed (`allow-scripts allow-same-origin`). The â€œopen in new tabâ€ button streams the generated markup via a `Blob` URL so the new page renders exactly what the iframe shows.

- **Copy / Download UX:**  
  Buttons disable when no code exists, display hover tooltips, and show inline confirmation badges. Downloads use an `.html` extension for easy inspection.

- **Syntax Highlighting:**  
  Switched from Ace Editor to Prism-based highlighting because Ace attempted to lazy-load resources blocked by CSP/integrity checks, causing blank renders. Prism works fully offline with bundler-friendly imports.
